## 1. 基本方針

### スライドの主役

この資料の主役は「技術」ではなく、利尻島の昆布干し現場で起きている判断の難しさと、それを減らす仕組みです。

コアメッセージ:

> 利尻の昆布は、利尻の天気で予報する。

補助メッセージ:

> 331か所の干場ごとに、雨・湿度・風・地形を見て、7日先まで「干せるか」を判断する。

### 見せ方の優先順位

1. 現場の課題
2. アプリでできること
3. 予報が信頼できる理由
4. GitHub / Render による継続運用
5. 今後の改善・協力依頼

### 避ける表現

- 「AIが全部判断します」のような過大表現
- 「絶対に当たる」などの断定
- 技術用語だけで押し切る構成
- 地域の経験や勘を否定する言い方

推奨表現:

- 「経験を補う判断材料」
- 「干す前に確認できる」
- 「場所ごとの差を見える化する」
- 「記録が増えるほど、利尻に合った予報へ育つ」

標準版は18枚。地域説明会なら12枚に短縮、技術説明会なら24枚に拡張する。

### 01. タイトル

タイトル: 利尻島昆布干場予報システム
サブタイトル: 331か所の干場ごとに、7日先の乾燥適性を予報する

画面:
- 利尻島、利尻富士、昆布干し、アプリ画面のいずれかを背景にする

話すこと:

### 02. 現場の課題

タイトル: 干せるかどうかの判断は、毎日の収入と品質に直結する

要点:
- 急な雨で昆布が濡れる
- 湿度が高く、思ったほど乾かない
- 利尻島内でも地区・海岸向き・風向きで条件が違う
- 短い漁期の一日を無駄にできない

画面:
- 昆布干し作業の写真
- 「雨」「湿度」「風」「場所差」の4アイコン

---

### 03. 解決策の一言

タイトル: 島の干場に、島の予報を。

要点:
- 利尻島の331干場を個別に見る
- 乾燥に必要な条件だけを表示する
- 前日夕方・当日早朝・急変時に知らせる

画面:
- アプリ画面の地図を大きく見せる
- 3つの通知時刻を小さく並べる

---

### 04. システム全体像

タイトル: 334地点の気象と記録をひとつの画面に集約

要点:
- 331干場
- アメダス沓形
- アメダス本泊
- 利尻山頂

画面:
- 利尻島の地図
- 干場点群 + 3つの特別地点を色分け

補足:
- 特別地点は乾燥記録の対象ではなく、検証・基準点として使う

---

### 05. 何を予報するのか

タイトル: 「天気」ではなく「昆布が乾く条件」を予報する

要点:
- 降水量: 0mmが絶対条件
- 最低湿度: 94%以下
- 平均風速: 2.0m/s以上
- 強風時は飛散注意として減点

画面:
- 3条件を大きくカード化
- 降水量、湿度、風速のシンプルなメーター

---

### 06. 実測データに基づく閾値

タイトル: 2025年の実測記録から、利尻向けの判定基準を作成

要点:
- H_1631_1434地点の21件記録
- 2025年6月19日から8月23日
- 完全乾燥9件、中止10件、部分乾燥2件
- 成功例は全て降水量0mm

画面:
- 成功 / 部分乾燥 / 中止の棒グラフ
- 閾値表を小さく配置

注意:
- 「現時点の実測データに基づく」と表現する

---

### 07. 地形補正

タイトル: Open-Meteoの5km予報を、干場スケールへ近づける

要点:
- Open-Meteo JMA MSMを利用
- 気温はOpen-Meteo側の標高補正を尊重
- 風速は森林・海岸条件で補正
- 湿度は水蒸気圧ベースで補正
- onshore windを利尻山頂からの方位で判定

画面:
- 「5kmメッシュ → 干場」のダウンスケーリング図
- 森林、海岸、山頂、風向の模式図

---

### 08. 地図UI

タイトル: 干場ごとの差を、地図で一目で確認

要点:
- 334地点マーカー
- 町・地区・部落で絞り込み
- 等値線で気温・湿度・風速の分布を可視化
- 利尻島伝統風名を表示

画面:
- `kelp_drying_map.html` のスクリーンショット
- フィルタ UI と地図を強調

---

### 09. 7日間予報

タイトル: 今日だけでなく、段取りに必要な7日先まで見る

要点:
- 干場ごとの7日間スコア
- 雨、湿度、風をまとめて判断
- 前日準備や作業調整に使える

画面:
- 7日間カードのスクリーンショット
- 良好 / 注意 / 中止の色分け

---

### 10. 通知システム

タイトル: 見に行かなくても、必要な時刻に知らせる

要点:
- 毎日16:00 JST: 翌日の予報
- 毎日01:30 JST: 当日の予報
- 1時間ごと: スコア急変時の緊急アラート
- 沖止め日やシーズン範囲を考慮

画面:
- スマホ通知のモック
- 16:00 / 01:30 / emergency の3段階タイムライン

画面:

## 5. ビジュアル方針

### 色

推奨:
- 海の青: 信頼、島、気象
- 昆布の深緑: 主題
- 白: 読みやすさ
- 警戒色の橙/赤: 注意、雨、強風

避ける:
- 紫系グラデーション中心のAIっぽい配色
- 暗すぎる画面
- 装飾過多の未来感

### 写真・図

優先素材:
- 利尻島 / 利尻富士
- 昆布干し作業
- アプリの実画面スクリーンショット
- 地図、点群、通知モック

図解:
- 334地点
- 3条件判定
- 5kmメッシュから干場スケールへの補正
- GitHub → Render → ユーザー の運用フロー
