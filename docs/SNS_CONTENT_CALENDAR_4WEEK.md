# SNS投稿カレンダー 初回4週間案

## 目的

ユーザーが内容確認してOKしやすいように、島内向けと全国向けを分けて、最初の4週間の投稿ペースと投稿案を整理する。

投稿前に必ず `AGENT_sns_fact_checker.md` で確認してください。

島内向け宣伝は9月末で一旦終了します。全国一次産業向けの発信は9月以降も継続します。

島内向け投稿では、週1本まで気象ミニ解説を入れてよいです。候補は `docs/RISHIRI_WEATHER_EDUCATION_POSTS.md` を参照してください。

## 島内向け投稿に定期的に入れる協力依頼

島内向け投稿では、強く押しつけない形で、定期的に干し記録の入力協力を依頼します。

```text
予報改善のため、干せた日・干せなかった日の記録入力にもご協力ください。
任意入力で、わかる範囲で大丈夫です。
```

投稿本文に毎回入れると重くなるため、以下の頻度を推奨します。

- Facebook/Instagram: 2週に1回
- LINE転送文: 月1回
- LP: 常時掲載

## 承認時の見方

ユーザーは各行について以下だけ見ればOKです。

- `title`
- `body`
- `scheduled_at_jst`
- `platform`
- `asset_notes`

問題なければ:

```text
status = approved
```

に変更してください。

ただし `note` と `line_share` は、最初は自動投稿ではなく下書き/転送文として扱うのがおすすめです。
