# docs/ フォルダ　ファイル管理インデックス

このフォルダは**外部に向けたドキュメント**（利用者・営業先など）を置く場所です。

開発者向けの技術仕様書（system_specification.md など）はルートディレクトリに置きます。

## ファイルの更新方法（上書き）

既存のファイルを最新内容で上書きします。新しいファイルは作りません。

1. 対象のファイルを開いて内容を書き直す（上書き保存）
2. ファイル内の「最終更新」日付を修正する
3. 上の「現在のファイル一覧」テーブルの「最終更新」列も同じ日付に修正する
4. 内容が大きく変わったときは、ファイル内の「更新履歴」テーブルにも1行追加する

---

## 新しい種類のドキュメントを追加するとき

全く新しい用途のドキュメントが必要になった場合のみ、新規ファイルを作ります。

1. このフォルダ（`docs/`）に `.md` ファイルを置く
2. 上の「現在のファイル一覧」テーブルに1行追加する

**例：**
```
| [NEW_FILE.md](NEW_FILE.md) | 新しいドキュメントの内容 | 対象者 | 2026-05-01 |
```

## フォルダの使い分け

```
（ルート）              開発者向けドキュメント
                        CLAUDE.md, system_specification.md など

docs/                   外部向けドキュメント（このフォルダ）
                        利用者・営業先に渡すもの

archive/deprecated/     用済みになったファイルの一時保管場所
```
