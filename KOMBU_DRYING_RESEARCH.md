## 1. 昆布の基本的な水分特性

### 1-1. 採取直後の水分量

| 状態 | 含水率（湿量基準） | 備考 |
|-----|----------------|------|
| 採取直後（生） | **86〜92%** | 種によって幅あり |
| 天日干し目標 | **14〜18%** | 出荷規格の目安 |
| 保存中の平衡 | **10〜16%** | 保管環境湿度に依存 |
| 微生物増殖限界 | aw < 0.60（含水率≒13%以下） | カビ・細菌の繁殖抑制 |

採取直後の昆布は含水量が重量の86〜92%を占め、これを14〜18%まで落とすのが乾燥の目標です。  
除去すべき水の質量は昆布乾燥重量の **約5〜7倍**に相当します。

### 1-2. 葉体の物理的特性

| 特性 | 値 | 備考 |
|-----|-----|------|
| 葉体厚さ | 2〜10 mm | 厚いほど乾燥に時間がかかる |
| 葉長 | 1〜5 m（収穫時） | 利尻昆布は比較的葉長が短い |
| 表面積/体積比 | 高い（薄板状） | 一般的な農産物より蒸発面積が広い |
| 表面粘液層（フコイダン） | 存在 | 最初の乾燥を若干遅らせる |

## 2. 乾燥の3段階プロセス

海藻の乾燥は食品工学的に次の3段階に分けられます（Constant Rate → Falling Rate）。

```
含水率
 ↑
90%┤ 採取直後
   │
   │  【第1段階：定率乾燥期（Constant Rate Period）】
   │   表面が常に湿潤状態を維持
   │   蒸発速度 = 表面への水分供給速度
   │   乾燥速度は温度・湿度・風速で決まる（内部抵抗なし）
   │   ← この段階が気象条件に最も敏感 →
   │
55%┤ ────── 臨界含水率（Critical Moisture Content）
   │
   │  【第2段階：減率乾燥期（Falling Rate Period）】
   │   内部の水分拡散が律速（表面が乾いてくる）
   │   乾燥速度は内部の有効水分拡散係数（Deff）に依存
   │   外部気象条件の影響が相対的に低下
   │
18%┤ ────── 乾燥目標含水率
   │
   │  【第3段階：平衡段階（Equilibrium Stage）】
   │   材料の水分活性 = 周囲の相対湿度
   │   それ以上乾燥しない（過乾燥防止の目安）
   ↓
```

**臨界含水率について（海藻類）**  
近縁種Gracilaria gracilis・Ulva lactucaの研究（ScienceDirect, 2024）では、臨界含水率は **55〜56%（湿量基準）** と報告されています。  
昆布（Laminaria/Saccharina japonica）においても同程度と推定され、**採取直後の約60%の水分が表面蒸発で除去**され、残り40%（含水率約35〜55%）から内部拡散律速に移行します。

**昆布干しとの対応**  
- **第1段階**（含水率90% → 55%）：気象条件が乾燥速度を支配 → **予報システムで最も重要な段階**
- **第2段階**（55% → 18%）：葉体内の水分移動が律速 → 気象条件の影響は相対的に小さい
- 実際の天日干しの「干せた/干せなかった」判断は主に第1段階の終了タイミングに依存

## 3. 乾燥速度モデル（薄層乾燥理論）

### 3-1. Pageモデル（昆布に最適）

昆布（Laminaria japonica）の乾燥動態を記述するモデルとして **Pageモデル** が最も精度が高いことが確認されています（Zhang et al. 2022; PMC9658940）。

```
MR = exp(−k × t^n)
```

| 記号 | 意味 | 単位 |
|-----|------|------|
| MR | 水分比（Moisture Ratio） = (M − Me)/(M0 − Me) | 無次元 |
| M | 時刻tの含水率 | % |
| M0 | 初期含水率 | % |
| Me | 平衡含水率 | % |
| k | 乾燥速度定数（温度・湿度・風速に依存） | s⁻¹ または min⁻¹ |
| n | 形状係数（通常 n > 1 で減率特性） | 無次元 |
| t | 乾燥時間 | s または min |

**モデルの精度**  
Pageモデルはヒートポンプ乾燥および太陽熱補助乾燥の両条件でR² = 0.873〜0.999 を達成し、MRの誤差は5%未満です。

### 3-2. 有効水分拡散係数（D_eff）

Fickの第2法則を薄板形状（昆布）に適用したときの見かけの拡散係数。

```
ln(MR) ≈ ln(8/π²) − (π² × Deff × t) / (4 × L²)
```

| 記号 | 意味 |
|-----|------|
| L | 葉体の半厚さ（m）。例: 厚さ4mm → L = 0.002 m |
| Deff | 有効水分拡散係数（m²/s） |

**Laminaria japonica の Deff 実測値**（Zhang et al. 2022）：

| 乾燥条件 | Deff 範囲 |
|---------|----------|
| ヒートポンプ乾燥（20〜50°C） | 1.00×10⁻¹¹ 〜 13.00×10⁻¹¹ m²/s |
| 太陽熱補助ヒートポンプ（S-HP） | 5.43×10⁻¹¹ 〜 11.32×10⁻¹¹ m²/s |
| ヒートポンプ単体（HP） | 1.04×10⁻¹¹ 〜 1.43×10⁻¹¹ m²/s |

→ 太陽放射が加わると Deff が**約7倍**向上します。これが天日干しで日射量が重要な理由の物理的説明です。

### 3-3. 活性化エネルギー（Ea）

乾燥速度定数 k の温度依存性はアレニウス式で表されます：

```
k = k0 × exp(−Ea / (R × T))
```

- Ea : 活性化エネルギー（J/mol）
- R : 気体定数（8.314 J/mol·K）
- T : 絶対温度（K）

昆布類の Ea は食品工学的には **15〜50 kJ/mol** の範囲が報告されており、これは温度が10°C上昇すると乾燥速度定数 k が **1.5〜2倍** 増加することを意味します。

### 3-4. 熱力学的パラメータ（Zhang et al. 2022 実測値）

| パラメータ | 記号 | 単位 | 意味 |
|-----------|-----|------|------|
| エンタルピー変化 | ΔH | kJ/mol | 水分除去に必要な熱エネルギー量 |
| エントロピー変化 | ΔS | J/mol·K | 乾燥システムの無秩序度変化 |
| ギブス自由エネルギー | ΔG | kJ/mol | 乾燥の自発性（ΔG < 0 で自発乾燥可） |

## 4. 気象条件と乾燥速度の関係

### 4-1. 温度の影響

**最も影響が大きい単一因子**（複数研究で一致）。

| 温度 | 相対的な乾燥速度 | 天日干しへの換算 |
|-----|--------------|--------------|
| 20°C | 基準（1.0） | 春・秋の低温日 |
| 25°C | 1.3〜1.5倍 | 初夏・初秋の適温日 |
| 30°C | 1.8〜2.2倍 | 夏季の晴天日（理想的） |
| 40°C | 3〜5倍 | ヒートポンプ条件 |
| 50°C | 5〜8倍 | 品質劣化リスクが高まる域 |

昆布における「温度vs乾燥速度」は **湿度の影響より有意に大きい** ことが研究で確認されています。

### 4-2. 相対湿度の影響

乾燥速度は飽和水蒸気圧と周囲水蒸気圧の差（水蒸気圧勾配）に比例します。

```
Rc ∝ (Ps(T_surface) − Pa) / R_boundary
```

| 相対湿度 | 水蒸気圧勾配 | 乾燥速度への影響 |
|---------|-----------|--------------|
| 20% | 最大（例: 25°C時 2.53−0.63 = 1.90 kPa） | 最速 |
| 50% | 中程度（同: 2.53−1.27 = 1.26 kPa） | 中程度 |
| 80% | 小（同: 2.53−2.02 = 0.51 kPa） | 遅い |
| 94% | 極小（同: 2.53−2.38 = 0.15 kPa） | 極めて遅い |
| ≥100% | 0以下 | **再吸湿**（乾燥逆転） |

**現行システムの閾値（最低湿度≤94%）との対応**  
湿度94%で水蒸気圧勾配はほぼゼロに近くなり、表面蒸発がほぼ停止します。これが実測データで **「湿度94%が乾燥成否の境界線」** として観測された物理的理由です。

### 4-3. 風速の影響

風速は**境界層の厚さ**を薄くすることで表面からの水蒸気輸送を促進します。

```
h_m ∝ U^0.5〜0.8  （強制対流の質量輸送係数）
```

| 風速 | 境界層の状態 | 乾燥速度への影響 |
|-----|-----------|--------------|
| 0 m/s（静穏） | 厚い境界層 → 拡散律速 | 最遅（表面に水蒸気が滞留） |
| 1.0 m/s | 薄くなり始める | 速度上昇 |
| 2.0 m/s | 境界層が十分薄い | **実測で成功可能な最低値** |
| 3〜5 m/s | 強制対流が支配的 | 急速乾燥（理想帯） |
| >8.5 m/s | 作業困難 | 干し台からの飛散リスク |

**2.0 m/sが最低閾値の物理的根拠**  
風速が2.0 m/s以下では昆布表面の大気境界層が十分厚く、水蒸気が境界層内に滞留することで局所的な飽和状態（高湿度）が形成されます。この境界層内の有効湿度が周囲の計測値より高いため、見かけの乾燥条件（湿度94%以下）が満たされていても実質的に乾燥が進まないケースが生じます。

### 4-4. 日射量（直達日射）の影響

日射は昆布表面を**直接加熱**し、葉体表面温度を周囲気温より高くすることで蒸発を促進します。

```
Q_solar → T_surface 上昇 → Ps(T_surface) 上昇 → 水蒸気圧勾配拡大 → Deff 増大
```

- 日射400 W/m²以上：D_effが約7倍向上（S-HP研究との比較より）
- 直達日射の有無が第1段階乾燥速度を**決定的に左右**
- 曇天（雲量>80%）では気温に関係なく乾燥がほぼ停止する場合あり

### 4-5. 4要因の相対的重要度（まとめ）

```
乾燥速度への影響の大きさ（研究知見より）:

温度 ≥ 日射 > 風速 >> 湿度（第1段階）
温度 > 風速 > 日射 > 湿度（第2段階、内部拡散律速時）
```

> **予報システム上の注意**: 現行の乾燥スコアは「最低湿度」と「風速」を主軸に置いていますが、
> 研究では**温度**と**日射量**が乾燥速度に与える影響が相対的に大きいことが示されています。

## 5. 天日干し（利尻島の現場作業）

### 5-1. 標準的な作業スケジュール（実地取材より）

| 時刻（JST） | 作業 | 気象条件の意味 |
|-----------|-----|------------|
| 02:30〜04:00 | 起床・漁に出発 | 前夜の露（朝露）の有無で晴天を判断 |
| 04:00〜08:00 | 昆布の水揚げ・干場への搬入 | — |
| 05:00〜08:00 | 干し台への展開（並べ上げ） | 朝露が残らないよう日の出直後から開始 |
| 09:00〜13:00 | 乾燥の主要段階 | 日射・気温・風速が最大になる時間帯 |
| 14:00〜16:00 | 乾燥確認・取り込み開始 | 夕方の結露前に回収 |
| 夕方 | 屋内へ取り込み | 夜間の再吸湿（結露）を防止 |

**「露が降りれば晴れ」の気象学的根拠**  
夜間の放射冷却が強く、地面温度が露点温度以下に下がる → 放射冷却が機能するほど空が晴れている → 翌日の晴天確率が高い。漁師の経験則は物理的に正しい指標です（ISLAND_METEOROLOGY_RESEARCH.md §7-1c 放射霧の節も参照）。

### 5-2. 干し方の実践的ルール

1. **葉を重ねない**: 重なり部分は乾燥できず品質格下げの原因
2. **葉側を上、根元側を下**: 乾燥の均一性を確保
3. **根元の向きを風向に合わせる**: 東風なら根元を東向き → 風が葉体に平行に流れ境界層を薄くする
4. **2日乾燥が原則**: 晴天続きでも1日では完全乾燥に達しないことが多い

### 5-3. 取り込みのタイミング

- **晴天日の取り込み時刻**: 概ね14:00〜16:00 JST
- **夕方の取り込み理由**: 16:00以降は海風が湿潤化（§5 海陸風参照）し、乾燥が逆転する可能性
- **雨・霧での緊急回収**: 含水率が急上昇し、等級が下がるため即時対応

### 5-4. 熟成（蔵乾燥）

初日の天日干しで含水率約18%まで乾燥後、さらに**蔵で1年間熟成**させることで最高品質の利尻昆布が完成します。  
熟成中に昆布は吸湿・放湿を繰り返し、グルタミン酸が熟成・凝縮されて「まろやかで深いだし」が形成されます。

## 6. 品質への影響—グルタミン酸・マンニトール

### 6-1. グルタミン酸（うま味成分）

| 指標 | 値 | 備考 |
|-----|-----|------|
| 昆布中のアミノ酸に占める割合 | 約80〜90% | グルタミン酸が支配的 |
| 生鮮昆布（Saccharina latissima）中のグルタミン酸 | 173 mg/g タンパク質 | 乾燥で有意に減少 |
| 高温乾燥（>50°C）による損失 | 有意な減少 | メイラード反応・熱分解 |
| 天日干し（<35°C）での保持 | 比較的良好 | 低温乾燥が推奨される理由 |

**高温乾燥の影響**  
50°C超の乾燥ではグルタミン酸の熱分解とメイラード反応（アミノ酸+糖の褐変反応）が顕在化します。昆布が「黒く」なる現象の一因です。

### 6-2. マンニトール（甘味・品質指標）

| 特性 | 内容 |
|-----|------|
| 化学的位置づけ | 糖アルコール（ソルビトール類縁） |
| 乾燥昆布表面の白い粉 | **マンニトール結晶**（カビではない） |
| 生育度合いの指標 | マンニトール量が多い → 十分に成熟した高品質昆布 |
| 乾燥速度との関係 | 天日干しで結晶析出が促進 → 白い粉が多い → 高品質の証 |

表面に白い粉が多く析出するには**緩やかな乾燥**（天日干し）が適しており、急速乾燥（高温機械乾燥）では結晶化が不十分になります。

### 6-3. 乾燥温度と品質指標の関係まとめ

| 乾燥温度 | グルタミン酸 | マンニトール | 色調 | 品質評価 |
|---------|------------|------------|------|---------|
| <35°C（天日） | 保持 | 析出良好 | 飴色〜緑褐色 | 最高 |
| 35〜50°C | やや低下 | 析出普通 | 緑〜褐色 | 良 |
| 50〜60°C | 低下 | 析出不十分 | 褐〜黒色 | 可 |
| >60°C | 有意に低下 | 劣化 | 黒色 | 不可 |

## 7. 天日干し vs 機械乾燥

### 7-1. 比較表

| 項目 | 天日干し | 低温機械乾燥（<40°C） | 高温機械乾燥（>50°C） |
|-----|---------|-------------------|-------------------|
| 乾燥時間 | 4〜5時間（第1日） + 2日間 | 4〜5時間 | 1〜2時間 |
| グルタミン酸保持 | 最良 | 良 | 劣る |
| マンニトール析出 | 最良 | 良 | 劣る |
| 昆布の色 | 飴色 | 良好 | 黒色 |
| 品質格付け | 一等品が出やすい | 同等可能 | 格下がり |
| 天候依存性 | 強（晴天必須） | なし | なし |
| コスト | 低（燃料不要） | 中（電力） | 中〜高 |
| 利尻産高級品 | **主流** | 補助的 | 非推奨 |

### 7-2. ハイブリッド手法（現代の実践）

天候リスクを下げるため、以下のハイブリッド乾燥が普及：

1. **午前中**: 天日干しで含水率を55%程度まで下げる（第1段階）
2. **午後**: 低温機械乾燥（35〜40°C）で18%まで仕上げ
3. 翌日以降: 天候次第でさらに天日干しまたは機械乾燥

これにより天日干しの品質メリットを活かしつつ、天候リスクを低減しています。

## 8. 平衡含水率と水分活性

### 8-1. 水分活性（aw）と安全性

```
aw = p / p0 = 相対湿度（小数）
```

| aw | 相対湿度 | 微生物リスク | 昆布の保存性 |
|----|---------|-----------|-----------|
| 0.95 | 95% | カビ・酵母増殖 | 数日以内 |
| 0.85 | 85% | 一部カビ | 短期 |
| 0.70 | 70% | 好乾性カビのみ | 中期 |
| 0.60 | 60% | 微生物増殖なし | 長期保存可 |
| 0.50 | 50% | 完全安全 | 1年以上 |

乾燥昆布の含水率14〜18%は、保管環境の相対湿度50〜65%で達成されるaw = 0.50〜0.65に相当します。

### 8-2. 再吸湿のリスク

保管環境の相対湿度が70%を超えると昆布が吸湿し、含水率が上昇してカビの発生リスクが高まります。  
これが「乾燥後の保管は密封・除湿が必須」と「夕方の取り込みが不可欠」な理由です。

### 8-3. 夜間再吸湿の定量的見積もり

利尻島の夏季夜間湿度は平均85〜95%に達します。  
仮に含水率18%まで乾燥した昆布を夜間外に放置すると：

- 夜間4〜8時間で含水率が **22〜30%まで上昇**する可能性
- 翌朝から再乾燥が必要になり、第1段階乾燥をやり直すことになる
- 品質等級の低下につながる

## 9. 現行予報システムへの示唆

### 9-1. 予報閾値の物理的根拠の整理

| 閾値 | 根拠（科学的） | 根拠（実測） |
|-----|------------|-----------|
| 降水量 = 0mm | 降雨で含水率が90%超に即座に回復 | 成功例21件すべてで0mm |
| 最低湿度 ≤ 94% | 相対湿度94%で水蒸気圧勾配≒0、蒸発停止 | 実測：94%まで成功、95%以上は困難 |
| 平均風速 ≥ 2.0 m/s | 大気境界層が薄くなり水蒸気輸送が始まる風速の下限 | 実測：2.0 m/sで成功例あり |
| 気温 ≥ 18.3°C | 低温では蒸発速度が低下（アレニウス依存） | 成功例の最低気温 |

### 9-2. 現行スコアでの不足点

#### (a) 日射量の過小評価

研究では**日射（直達日射量）が乾燥速度に対して温度と同程度の影響を持つ**ことが示されています（Deff が日射400 W/m²以上で7倍向上）。  
現行スコアでは `solar_radiation` データは取得済みですが、乾燥スコアへの反映が弱い可能性があります。

**改善提案**:
```python
# calculate_enhanced_drying_score() への追加例
if solar_radiation is not None:
    if solar_radiation >= 400:  # 強日射
        score += 15
    elif solar_radiation >= 200:  # 中程度
        score += 8
    elif solar_radiation < 50:  # 曇天・ほぼ日射なし
        score -= 10
```

#### (b) 乾燥速度の時間帯依存性のモデル化

昆布乾燥は午前中（定率乾燥期、第1段階）と午後（減率乾燥期、第2段階）で支配因子が異なります。

| 時間帯 | 支配因子 | 現行重み付け | 研究の示唆 |
|-------|---------|-----------|---------|
| 04〜10時 | 風速・湿度（境界層） | 風70%・日射30% | ほぼ適切 |
| 10〜16時 | 日射・気温（加熱） | 風40%・日射60% | 温度をもっと重視すべき |

#### (c) 再吸湿リスク（夕方の湿度上昇）

現行の予報は1日の乾燥可否を単純スコアで判定しますが、**午後の湿度上昇（海風によるもの）** が乾燥を逆転させるリスクを加味していません。

**追加指標（案）**:
```
夕方湿度上昇スコア = (RH_16時 - RH_10時) / 時間
→ 上昇が急激（>3%/時）の場合、取り込み推奨通知を早める
```

#### (d) 乾燥完了時間の予測

現行の「乾燥時間予測」は段階別スコアから経験的に算出されていますが、  
Pageモデルを適用した理論的予測に高度化できます。

```python
# 簡易Pageモデルによる天日干し完了時間予測（疑似コード）
def estimate_drying_time(temp_c, humidity_pct, wind_ms, radiation_wm2):
    # 温度依存の乾燥速度定数（仮パラメータ）
    k_base = 0.0003  # min⁻¹ @ 25°C
    n = 1.2          # 形状係数

    # 温度補正（アレニウス近似）
    k = k_base * (1.5 ** ((temp_c - 25) / 10))

    # 風速補正（境界層効果）
    k *= max(0.3, min(2.0, wind_ms / 3.0))

    # 日射補正
    k *= max(0.5, min(2.5, radiation_wm2 / 300))

    # 湿度補正（水蒸気圧勾配）
    k *= max(0, (100 - humidity_pct) / 50)

    # MR = 0.10（含水率 ≈18%に相当）となる時刻を求める
    # exp(-k * t^n) = 0.10 → t = (-ln(0.10)/k)^(1/n)
    import math
    t_minutes = ((-math.log(0.10)) / k) ** (1 / n)
    return t_minutes
```

### 9-3. 「完全乾燥」と「部分乾燥」の含水率的解釈

| 記録 | 含水率の推定 | 乾燥段階 |
|-----|-----------|---------|
| 完全乾燥 | ≤18%（目標達成） | 第2段階完了 |
| 部分乾燥（干したが完全には…） | 20〜30%程度 | 第2段階途中で日没 |
| 中止 | >55%（第1段階も終わらず） または 降雨 | 第1段階で断念 |

### K1 — `calculate_enhanced_drying_score()` に日射スコアが存在しない

**現行コード** (`start.py` L968-996):
```python
def calculate_enhanced_drying_score(temp_max, humidity, wind_speed, precipitation, lat, lon):
    score = 0
    if temp_max > 15: score += 25
    if temp_max > 20: score += 15
    if humidity < 70: score += 20
    if humidity < 60: score += 10
    if wind_speed > 1.5: score += 15
    if wind_speed > 3:   score += 10
    if precipitation < 1: score += 15
    # ... 地形補正のみ
```

**研究の知見**:  
`Deff` は日射 ≥ 400 W/m² で **約7倍**向上する（§3-2）。研究者間の一致した見解では「温度 ≥ 日射 > 風速 >> 湿度」が乾燥速度への影響順位（§4-5）。日射を全く含まないこの関数は、曇天と晴天を区別できません。

**問題の大きさ**:  
この関数の結果が `drying_score` キーとして保存され、`forecast_history/` に記録されます。将来の精度検証でも使われる重要なスコアであり、日射なしでは晴天日の予報値が過小評価されます。

**改善提案**:
```python
def calculate_enhanced_drying_score(temp_max, humidity, wind_speed, precipitation,
                                    lat, lon, avg_solar_radiation=None):
    score = 0
    if temp_max > 15: score += 20
    if temp_max > 20: score += 10
    if temp_max > 25: score += 10   # 高温域を細分化

    if humidity < 70: score += 15
    if humidity < 60: score += 10

    if wind_speed >= 2.0: score += 15  # 境界層閾値
    if wind_speed > 3.0:  score += 10

    if precipitation == 0: score += 15  # 絶対条件

    # 日射スコア（K1 修正）
    if avg_solar_radiation is not None:
        if avg_solar_radiation >= 600:   score += 20
        elif avg_solar_radiation >= 400: score += 15  # Deff 7倍の閾値
        elif avg_solar_radiation >= 200: score += 8
        elif avg_solar_radiation < 50:   score -= 15  # 曇天ペナルティ
    # ... 地形補正
```

> **実装メモ**: `avg_solar_radiation` は `get_forecast()` 内の `hourly` データから算出可能。  
> `daily` APIから `shortwave_radiation_sum` を追加取得することでも対応できます。

### K2 — 気温スコアの段階関数が粗すぎる

**現行コード**:
```python

if temp_max > 20: score += 15
```

20°Cと29°Cが同じスコアになります。

**研究の知見**:  
アレニウス式より、温度10°C上昇で乾燥速度定数 k が **1.5〜2倍** 増加（§3-3）。25°Cと30°Cでは約1.3倍の差があります。

**改善提案**（連続的な気温重みへの変更）:

```python
# アレニウス近似（基準温度25°C）
import math
if temp_max is not None:
    temp_factor = 1.5 ** ((temp_max - 25) / 10)  # 10°Cで1.5倍
    temp_score = min(40, max(0, int(temp_factor * 20)))
    score += temp_score
```

> **優先度**: 中。既存スコアの温度項（最大+40点）はほぼ正しい方向だが、精密化で信頼性が向上します。

### K3 — 段階重み（換気60%/加熱40%）と研究知見の不一致

**現行コード** (L4197-4200):
```python
wind_weight  = 0.6  # 換気期（4:00-10:00）
solar_weight = 0.4  # 加熱期（10:00-16:00）
overall_score = ventilation_score * wind_weight + heat_supply_score * solar_weight
```

**研究の知見**:  
乾燥速度の影響順位は「温度 ≥ 日射 > 風速 >> 湿度」（§4-5）。風速（換気）を最優先にする60/40配分は第1段階（定率乾燥期）の特性を反映していますが、定率乾燥期（含水率90%→55%）が終わった後は温度・日射が支配的になります。

**現行の合理性**:  
コメント（L4192-4196）に「初期段階の表面乾燥が重要」という実測データ基準の根拠が記録されています。これは第1段階（表面蒸発律速）の重要性と一致しており、現行の60/40が完全に誤りではありません。

**改善案（日射加重型）**:  
加熱期スコアの内部重みを見直すことで対応可能です:
```python
# 現行: (temp_score * 0.6 + solar_score * 0.4) × humidity_factor
# 改善: 研究知見に合わせて温度と日射をほぼ同等に
hour_score = (temp_score * 0.5 + solar_score * 0.5) * humidity_factor
```
段階間の60/40はそのままで、加熱期内部の配分だけ調整する保守的な改善です。

### K5 — 湿度係数の線形近似（水蒸気圧勾配は非線形）

**現行コード** (L4144, L4184):
```python
humidity_factor = max(0, (100 - humidity) / 100)  # 線形
```

**研究の知見**:  
蒸発速度は水蒸気圧勾配 `Ps(T) - Pa` に比例します（§4-2）。飽和水蒸気圧はクラウジウス・クラペイロン式に従い指数的に増加するため、同じ湿度変化でも高湿度域（85〜94%）では勾配の減少が線形より急激です。

| 湿度 | 線形係数 | 水蒸気圧勾配（25°C） | 実際の乾燥効率比 |
|-----|--------|----------------|--------------|
| 60% | 0.40 | 1.27 kPa | 基準 |
| 80% | 0.20 | 0.51 kPa | 0.40（線形: 0.50） |
| 94% | 0.06 | 0.15 kPa | 0.12（線形: 0.06） |

線形近似は高湿度側で過大評価、中湿度側で過小評価になります。

**改善提案（非線形変換）**:
```python
import math

def vapor_pressure_gradient_factor(humidity_pct, temp_c=20):
    """水蒸気圧勾配ベースの非線形湿度係数"""
    # マグナス公式による飽和水蒸気圧
    es = 6.112 * math.exp(17.67 * temp_c / (temp_c + 243.5))
    ea = es * (humidity_pct / 100)
    gradient = max(0, es - ea)
    # 湿度20%時の勾配で正規化
    es_ref = 6.112 * math.exp(17.67 * temp_c / (temp_c + 243.5))
    max_gradient = es_ref * (1 - 0.20)
    return min(1.0, gradient / max_gradient)
```

> **優先度**: 低〜中。既存の線形近似でも実測データ（21件）との整合は保てており、精密化は精度向上への貢献が小さい可能性もあります。

### K6 — 午後の再吸湿リスク（湿度上昇率）が未算出

**現行の状況**: 1日を通じた最低湿度・平均湿度は計算されていますが、時系列の変化トレンドは評価されていません。

**研究の知見**:  
利尻島の夏季、午後14:00〜16:00は海風の強化で湿度が急上昇します（§5-3、ISLAND_METEOROLOGY_RESEARCH.md §5）。含水率が18%まで下がった昆布でも、湿度が急上昇すると夜間に22〜30%まで再吸湿します（§8-3）。

**改善提案**:
```python
def calculate_remoistening_risk(hourly_data):
    """取り込み前の再吸湿リスク算出"""
    # 10時〜16時の湿度列
    afternoon_humidity = [
        h['humidity'] for h in hourly_data
        if h.get('humidity') is not None and 10 <= int(h['time'].split(':')[0]) <= 16
    ]
    if len(afternoon_humidity) < 2:
        return {'risk': 'unknown', 'rate': None}

    rise_rate = (afternoon_humidity[-1] - afternoon_humidity[0]) / len(afternoon_humidity)  # %/時

    if rise_rate > 4.0:
        return {'risk': 'high',   'rate': rise_rate, 'advice': '14:00以前の早期取り込みを推奨'}
    elif rise_rate > 2.0:
        return {'risk': 'medium', 'rate': rise_rate, 'advice': '16:00までに取り込み'}
    else:
        return {'risk': 'low',    'rate': rise_rate, 'advice': '通常通り取り込み可'}
```

> **活用**: `stage_analysis` に `remoistening_risk` を追加し、フロントエンドの取り込み推奨時刻表示に連携させる。

### K7 — 乾燥完了時間予測がPageモデルを使わない経験的ステップ

**現行コード** (L4231-4245):
```python
if overall_score >= 85:   completion_time = "14:00頃完全乾燥"
elif overall_score >= 70: completion_time = "16:00頃完全乾燥"
elif overall_score >= 55: completion_time = "18:00頃完全乾燥"
elif overall_score >= 40: completion_time = "夕方までに部分乾燥"
else:                      completion_time = "乾燥困難、延期推奨"
```

**研究の知見**:  
Pageモデル `MR = exp(−k × t^n)` により、気象条件から理論的な乾燥時間を算出できます（§3-1）。スコアのしきい値から機械的に時刻を返す現行方式より物理的根拠が強い予測が可能です。

**改善提案** (`§9-2 (d)` の疑似コードを精密化):
```python
def estimate_drying_completion_hour(temp_c, humidity_pct, wind_ms, radiation_wm2,
                                    start_hour=5, target_mr=0.10):
    """
    Pageモデルによる天日干し完了時刻（JST）の推定
    target_mr=0.10: MR≈0.10 → 含水率≈18%（目標含水率）
    """
    import math
    k_base = 0.0003   # min⁻¹ @ 25°C（Laminaria japonica 推定値）
    n = 1.2           # 形状係数（減率乾燥特性）

    # 温度補正（アレニウス近似、活性化エネルギー=25 kJ/mol相当）
    k = k_base * (1.5 ** ((temp_c - 25) / 10))

    # 風速補正（境界層効果）
    k *= max(0.3, min(2.0, wind_ms / 3.0))

    # 日射補正（Deff向上効果）
    if radiation_wm2 >= 400:   k *= 2.0
    elif radiation_wm2 >= 200: k *= 1.3
    elif radiation_wm2 < 50:   k *= 0.4

    # 水蒸気圧勾配補正
    k *= max(0.01, (100 - humidity_pct) / 50)

    if k <= 0:
        return None, None  # 乾燥不可

    t_minutes = ((-math.log(target_mr)) / k) ** (1 / n)
    completion_hour = start_hour + t_minutes / 60
    return round(completion_hour, 1), round(t_minutes, 0)
```

> **優先度**: 低。ステップ予測で実用上の問題はないが、研究的な根拠のある予測に置き換えることで将来の精度評価に役立ちます。

### 現行実装の評価（強み）

上記ギャップは改善余地を示しますが、現行実装の以下の設計は研究知見と良く合致しています：

- **2段階評価（換気期/加熱期）の考え方**: 定率乾燥期（表面蒸発）と減率乾燥期（内部拡散）の分離に対応
- **実測21件に基づく閾値**: 理論ではなく現地データからの帰納的設計
- **最低湿度94%・風速2.0m/sの物理的説明**: 水蒸気圧勾配・境界層理論と整合（本ドキュメント§4-2、§4-3）
- **PWV（可降水量）とPBLH（境界層高度）の組み込み**: 高度な気象指標の活用
- **dewpoint_2m データ取得済み**: 霧リスク算出（temp - dewpoint < 2°C）への即座の活用が可能
