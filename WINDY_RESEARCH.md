## 1. Windyとは

Windyは **チェコ発祥の気象可視化アプリ**（Web・iOS・Android対応）で、風・波・天気を地図上のリアルタイムアニメーションで表示することを主軸とした、パイロット・漁師・船乗り・アウトドア愛好家向けの高精度気象予報ツールです。

| 項目 | Windy.com | Windy.app |
|-----|---------|----------|
| 運営 | Windyty社（チェコ） | 別会社（Windy Weather World Inc.） |
| 主な強み | 地図アニメーション・高精度モデル | 海洋特化・釣り・ソルナー理論 |
| 料金 | 無料（プレミアムあり） | 無料（Pro版あり） |
| 利用者 | パイロット・漁師・気象マニア | マリンスポーツ・釣り人 |

**漁師の利用実態（日本）**  
日本の沿岸漁業従事者は Windy.com を以下の目的で活用しています：
- 出漁判断（風速・波高・うねりの確認）
- 「風裏ポイント」探し（地図上の風向で海況を把握）
- 黒潮位置の確認（海水温レイヤー）
- 天候急変の把握（レーダー・雷検知）

## 2. 主要機能一覧（カテゴリ別）

## 3. 漁師・昆布漁業者に特に有用な機能

### 3-1. 出漁判断（海況確認）

```
最重要の確認順序（漁師の典型的な使い方）:

① 波高・うねりレイヤー → 沖での作業安全性
② 風パーティクルアニメーション → 風裏ポイントの視覚的把握
③ リアルタイム雨レーダー → 出漁後の天候急変リスク
④ 雷検知レイヤー → 落雷リスクの有無
⑤ 潮汐グラフ → 昆布が露出する時間帯（低潮時）
```

### 3-2. 昆布干し作業における潜在的活用

| Windy機能 | 昆布干しへの活用 |
|---------|--------------|
| 海水温（SST）レイヤー | 冷水塊（親潮）接近 → 霧発生リスクの予測（当アプリで未実装） |
| 気象前線表示 | 数日先の前線通過タイミングを把握し、干し始め時期を最適化 |
| CAPE指数 | 急発達する積乱雲（にわか雨）の出現リスク評価 |
| リアルタイム雨レーダー | 接近する雨域を事前に把握し、緊急取り込みの判断を早める |
| 潮汐予報 | 干潮時刻を把握し、朝の干し台展開時刻を調整 |
| ソルナー理論 | 月相別の昆布の成熟度・品質への影響（仮説的） |
| 偏差指数（予報信頼度） | 予報の確実性を数値で確認し、長期計画の信頼性を判断 |

### 3-3. Windyの UI/UX 設計の特長

- **地図が主役**: すべての情報を地図上で直感的に確認できる（数値より視覚優先）
- **時間スライダー**: 地図上のアニメーションを時間軸で操作
- **ピッカー機能**: 地図の任意地点をクリックすると即座に詳細予報ポップアップ
- **スポットお気に入り登録**: 毎回座標を入力せず登録地点に即アクセス
- **モデル切替の容易さ**: ワンタップで ECMWF/GFS を切り替えて不確実性を確認

## 4. 予報モデルとデータソース

### 4-1. Windy.comで利用可能な予報モデル

| モデル名 | 機関 | 解像度 | 特徴 |
|---------|-----|--------|------|
| **ECMWF** | 欧州中期予報センター | 9km | 中期予報の世界最高精度。特に降水・雲に強い |
| **GFS** | NOAA（米国） | 13km | 全球モデル。無料アクセス可能で広く利用 |
| **ICON** | DWD（ドイツ） | 13km | 欧州での高精度。高解像度ネスティング版あり |
| **AROME** | Météo-France | 1.3km | 欧州ローカル高解像度 |
| **NAM** | NOAA | 12km | 北米ローカルモデル |
| **HRRR** | NOAA | 3km | 米国高解像度リアルタイム |
| **Meteoblue** | スイス | 30km | 気候・長期予測に特化 |

**当アプリとの比較**:  
現行システムは Open-Meteo 経由で JMA MSM（5km）と GSM（13km）を使用。ECMWFやICONなど複数モデルの比較機能はありません。

### 4-2. 海洋データソース

| データ | ソース |
|-------|-------|
| 潮汐予報 | WorldTides.info（全球検潮所データ） |
| 海流・潮流 | CMEMS（コペルニクス海洋環境監視サービス） |
| 海面水温（SST） | 衛星リモートセンシング（NOAA/ESA） |
| 波浪モデル | NOAA WaveWatch III + ECMWF Wave |

## 6. 導入候補機能と実装提案

### W2 — リアルタイム雨レーダー（雨ナウキャスト）

**優先度**: 🔴 高

**背景**:  
降雨が確認された時点で昆布の緊急取り込みが必要になります（含水率が即座に90%超に回復）。現行のOpen-Meteo予報は1時間単位ですが、雨レーダーなら **現在進行中の雨域** を分単位で確認できます。

**実装案**:
```html
<!-- 気象庁ナウキャスト（雨レーダー）の公式タイル埋め込み -->
<!-- JMA提供の高解像度降水ナウキャスト（5分更新、1km解像度）を使用 -->
<!-- 非商用なら無料で利用可能 -->

// Leaflet.js での実装例
const jmaRadarLayer = L.tileLayer(
    'https://www.jma.go.jp/bosai/jmatile/data/nowc/{time}/{z}/{x}/{y}.png',
    {
        attribution: '気象庁ナウキャスト',
        opacity: 0.6,
        tms: false
    }
);
```

**代替**: Open-Meteo は `precipitation_probability` を提供しており、これを時系列表示するだけでも現行より改善できます（→ W11 と組み合わせ）。

### W5 — 潮汐予報

**優先度**: 🔴 高

**背景**:  
利尻島での昆布漁は干潮時に岩場が露出することで採取可能になる地点があります。また朝の干し台展開（05:00〜08:00）は満潮・干潮の時刻と連動して作業効率が変わります。

**データソース**:
- **気象庁潮位表 API**（無料）: 日本国内の主要港湾の潮汐予報
  - 沓形港は気象庁潮位観測点に該当する可能性あり
- **WorldTides.info API**（有料、Windyが採用）: 全球対応
- **Open-Meteo はサポートなし**（非対応）

**実装案**:
```python

# 最寄り潮位観測点：稚内（稚内港）または留萌
# JMA潮位CSV形式で提供（無料）
import requests

def get_tide_forecast(station_id='350010', date_str='20260504'):
    """
    気象庁 潮位観測データAPI
    station_id: 稚内=350010, 留萌=480010 など
    """
    url = f'https://www.data.jma.go.jp/gmd/kaiyou/data/db/tide/suisan/{date_str}/{station_id}.csv'
    response = requests.get(url, timeout=10)
    # ... CSV解析
```

**フロントエンド表示案**:
```
干場カードへの追加表示（例）:
  🌊 潮汐  今日: 満潮 06:23 (132cm) / 干潮 12:47 (18cm)
           明日: 満潮 07:15 (128cm) / 干潮 13:38 (22cm)
```

### W10 — CAPE指数（対流有効位置エネルギー）

**優先度**: 🟡 中

**背景**:  
CAPE が高いと積乱雲が急発達し、予報にない突発的な強雨が発生します。昆布干し中の急な雨は緊急取り込みを迫ります。現行アプリにはCAPEが取得されていませんが、**Open-Meteoは `cape` 変数を提供**しており、APIパラメータ追加のみで取得できます。

**実装コスト**: 極小（APIパラメータ1つ追加）

**実装案**:
```python
# start.py の Open-Meteo URL に追加するだけ
# 既存のhourlyリストに 'cape' を追加
url = (
    f"...&hourly=...,cape"
)

# スコアへの反映（assess_drying_risk に追加）
def assess_cape_risk(cape_value):
    """対流不安定リスク評価"""
    if cape_value is None:
        return {'risk': 'unknown', 'score_penalty': 0}
    if cape_value > 1000:   # 強い不安定
        return {'risk': 'high',   'score_penalty': -20, 'warning': '突発的な強雨リスク高'}
    elif cape_value > 500:  # 中程度
        return {'risk': 'medium', 'score_penalty': -10, 'warning': '積乱雲発達リスクあり'}
    elif cape_value > 200:  # 弱い不安定
        return {'risk': 'low',    'score_penalty': -3,  'warning': 'にわか雨注意'}
    else:
        return {'risk': 'none',   'score_penalty': 0}
```

**フロントエンド表示案**:
```
乾燥スコアカードへの追加:
  ⚡ CAPE: 350 J/kg → ⚠️ 積乱雲・にわか雨に注意
```

### W11 — 降水確率（PoP: Probability of Precipitation）

**優先度**: 🟡 中

**背景**:  
現行アプリは「降水量（mm）」のみ表示していますが、「降水確率（%）」は意思決定に異なる情報を提供します。

```
例:
- 「降水量 0.2mm、確率 80%」= 高確率で雨が来るが少量
- 「降水量 0.5mm、確率 15%」= ほぼ降らないが降れば昆布干し中止
```

**データソース**: Open-Meteo は `precipitation_probability` を既に提供しており、API追加コスト=ゼロ。

**実装案**:
```python
# Open-Meteo URLに追加
hourly_params = [
    ...,
    'precipitation_probability'   # 0-100%
]

# 乾燥スコアへの反映
def apply_pop_risk(score, pop_value):
    """降水確率によるスコア調整"""
    if pop_value is None:
        return score
    if pop_value >= 70:
        return score * 0.3   # 70%以上: 大幅減点
    elif pop_value >= 50:
        return score * 0.6   # 50-70%: 中程度減点
    elif pop_value >= 30:
        return score * 0.85  # 30-50%: 軽微減点
    return score             # <30%: 変更なし
```

**フロントエンド**: `乾燥スコア: 72 / 🌧 降水確率: 35%` のように並記表示。

### W6 — 海水温（SST）連携

**優先度**: 🟡 中

**背景**:  
利尻島の霧は親潮（冷水塊）の接近と密接に関連しています（ISLAND_METEOROLOGY_RESEARCH.md §7）。海水温が15°C以下に下がると海霧の発生リスクが急上昇します。現行の霧リスク評価は露点降下量から推定していますが、SSTを加えることで海洋由来の霧を直接評価できます。

**データソース**:
- **Open-Meteo 海洋API** (`marine-api.open-meteo.com`): 波高、うねりに加えSST取得可能
- 利尻島近海の座標: 45.18°N, 141.18°E

**実装案**:
```python
def get_sea_surface_temperature(lat, lon):
    """Open-Meteo Marine API からSST取得"""
    url = (
        f"https://marine-api.open-meteo.com/v1/marine"
        f"?latitude={lat}&longitude={lon}"
        f"&daily=sea_surface_temperature"
        f"&timezone=Asia/Tokyo"
    )
    response = requests.get(url, timeout=10)
    data = response.json()
    sst = data['daily']['sea_surface_temperature'][0]  # 今日のSST

    # 霧リスク評価（気温 - SST の差）
    return sst

def assess_fog_risk_from_sst(sst, air_temp):
    """海水温から霧リスクを評価"""
    if sst is None or air_temp is None:
        return 'unknown'
    sst_air_diff = air_temp - sst
    if sst < 10:
        return 'very_high'   # 冷水塊接近: 極めて高い霧リスク
    elif sst < 15:
        return 'high'        # 親潮影響: 高い霧リスク
    elif sst_air_diff > 5:
        return 'medium'      # 暖気移流 + 冷海面: 中程度
    return 'low'
```

### W9 — ソルナー理論（月相・潮汐と昆布品質の関係）

**優先度**: 🟢 低〜中

**背景**:  
ソルナー理論はもともと釣りの「魚の活性」予測ですが、昆布の分野でも「月齢と成長・品質の関係」は古くから経験則として語られています。また**漁師が Windy に期待する機能**の一つであることから、当アプリに取り込むことでユーザー体験が向上します。

**実装案（ゼロ外部APIで実装可能）**:
```python
import math
from datetime import datetime, timezone

def calculate_moon_phase(date):
    """月齢計算（簡易実装）"""
    # 2000年1月6日 18:14 UTC = 新月
    new_moon_epoch = datetime(2000, 1, 6, 18, 14, tzinfo=timezone.utc)
    lunar_cycle = 29.530588853  # 朔望月（日）
    dt = (date - new_moon_epoch).total_seconds() / 86400
    phase = (dt % lunar_cycle) / lunar_cycle  # 0.0=新月, 0.5=満月
    return phase

def get_solunar_score(date, lat, lon):
    """
    ソルナー指数（0-100）
    新月・満月の前後2日: 魚の活性・昆布採取の伝統的な好機
    """
    phase = calculate_moon_phase(date)
    # 新月（phase≈0, 1）または満月（phase≈0.5）付近で高スコア
    proximity_to_new = min(phase, 1 - phase)        # 0〜0.5
    proximity_to_full = abs(phase - 0.5)             # 0〜0.5
    best_proximity = min(proximity_to_new, proximity_to_full)  # 0=新/満月
    solunar_score = max(0, 100 - best_proximity * 400)
    return round(solunar_score)
```

**表示案**:
```
🌕 月相インデックス: 82 / 100（満月2日後 — 昆布採取の好機）
```

### W13 — 複数予報モデル比較（予報の不確実性表示）

**優先度**: 🟢 低

**背景**:  
「明日は本当に晴れるのか？」を判断するとき、モデル間の一致度が信頼性の指標になります。ECMWFとGFSが同じ予報なら確信度が高く、食い違っているなら「様子見」が合理的です。

**実装案**:  
Open-Meteo は `models` パラメータで複数モデル同時取得が可能です。

```python
# JMA MSM + ECMWFの両方を取得して比較
url_ecmwf = f"...&models=ecmwf_ifs025&..."

# 両モデルの乾燥スコアを計算して差を表示
score_jma   = calculate_enhanced_drying_score(...)
score_ecmwf = calculate_enhanced_drying_score(...)
confidence  = max(0, 100 - abs(score_jma - score_ecmwf) * 2)
```

**表示案**:
```
乾燥スコア: 68（予報信頼度: 🟡 中程度 — JMAとECMWFで12点の差）
```
