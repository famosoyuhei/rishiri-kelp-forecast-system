## チェックリスト

### A. エマグラム図の描画

- [ ] 気温線（温度断面）が正しい気圧面（1000/850/700/500 hPa等）に描画されるか
- [ ] 露点線が気温線と区別できる色・線種で描画されるか
- [ ] 気温と露点の差が2°C未満の高度帯（霧・雲リスク域）が視覚的に強調されているか
- [ ] 対流圏内の温度減率（標準: 6.5°C/km）と実際の減率を比較できるか
- [ ] 風の矢羽根（または矢印）が各気圧面の高度に正しく表示されるか
- [ ] 縦軸（気圧 hPa）と横軸（温度 °C）のスケールが正しく表示されるか

### B. データ取得とフォールバック

- [ ] Open-Meteo APIから高層データ（850/700/500 hPa）が正常に取得されるか
- [ ] API取得失敗時にエラーが画面に表示され、空のままにならないか
- [ ] 利尻山頂（R_1800_2392、標高1,721m）を選択したとき、地上付近が850hPa相当になるか
- [ ] 予報時刻の切り替え（今日/明日/〜7日後）でエマグラムが正しく更新されるか

### C. 初心者向け解説・UI

- [ ] エマグラムの読み方の簡易ガイドがタブ内またはツールチップで提供されているか
- [ ] 「乾燥に良い大気状態」の判定結果（乾燥・普通・湿潤など）がテキストで表示されるか
- [ ] 昆布干しに関連するキーワード（逆転層・霧リスクなど）が初心者にわかる言葉で説明されているか
- [ ] スマホ縦画面でエマグラム図が横スクロールなしに閲覧できるか（またはスクロール誘導がわかりやすいか）

### D. 科学的整合性

- [ ] Open-Meteoから取得した気温がCLAUDE.mdの仕様どおりOpen-Meteo側で標高補正済みか（二重補正がないか）
- [ ] 利尻山頂（1,721m）のデータが地表面（14m）のデータと矛盾していないか
- [ ] 850hPaの気温とアメダス観測データ（沓形）との差が物理的に妥当な範囲か
