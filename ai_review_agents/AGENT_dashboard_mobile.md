## チェックリスト

### A. アプリロゴ（app_icon.png）の統一（v2.6.0新機能）

- [ ] `dashboard.html` にapp_icon.pngが表示されているか
- [ ] `mobile_forecast_interface.html` にapp_icon.pngが表示されているか
- [ ] `kelp_drying_map.html` にapp_icon.pngが表示されているか
- [ ] 3ファイルで同じ画像パス・同じサイズ基準で統一されているか
- [ ] app_icon.pngが存在しない場合にalt属性や代替テキストが設定されているか

### B. ダッシュボード（dashboard.html）

- [ ] メイン画面と同じAPIエンドポイントからデータを取得しているか（データの乖離がないか）
- [ ] 334地点の概況（今日何地点が「干せる」か）が表示されているか
- [ ] 利尻町 / 利尻富士町別の集計が表示されているか
- [ ] 乾燥記録の入力・確認機能がある場合、動作するか
- [ ] 最終データ更新時刻がJSTで表示されているか

### C. モバイル画面（mobile_forecast_interface.html）

- [ ] iPhone SE（375px）・iPhone 14（390px）・Android（360px）等の幅で表示崩れがないか
- [ ] タッチターゲット（ボタン・リンク）が44px以上の高さを確保しているか
- [ ] 干場の乾燥スコアが大きな文字・アイコンでひとめで把握できるか
- [ ] スワイプ操作（日付切り替えなど）が期待どおり動作するか
- [ ] フォントサイズが14px以上で老眼でも読みやすいか

### D. メイン画面との一貫性

- [ ] スコア計算ロジックがkelp_drying_map.htmlと同一か（同じAPIを使っているか確認）
- [ ] 利尻島伝統風名の表示がある場合、rishiri_wind_names.jsと同じ内容か
- [ ] 色コーディング（スコア高=緑、低=赤など）がメイン画面と一致しているか
- [ ] 「干場」「記録」「通知」などの用語がメイン画面と統一されているか

### E. ナビゲーション・遷移

- [ ] dashboard.htmlからkelp_drying_map.htmlへのリンクが機能するか
- [ ] mobile_forecast_interface.htmlからメイン画面へ戻れるか
- [ ] ブラウザの「戻る」ボタンで正しく前の画面に戻れるか
