## チェックリスト

### A. 通知文言のLINE転送適性

**夕方通知（16:00 JST）の文言:**
- [ ] 「明日○月○日は△△干場で干せます」のように日付・干場名が明示されているか
- [ ] スコアの数値だけでなく「◎ 乾燥適」「△ 微妙」「× 不適」などの日本語判定が含まれるか
- [ ] 文字数が120文字以内でLINEのプレビューに収まるか
- [ ] 絵文字が使われている場合、Android/iPhoneどちらでも文字化けしないものか（推奨: ☀️🌧️💨など基本絵文字）

**早朝通知（01:30 JST）の文言:**
- [ ] 「今日○月○日の干し場予報」と日付が明示されているか
- [ ] 「何時頃まで干せる」「風向は○○（○○風）」などの具体情報があるか
- [ ] 「昆布漁師向け」らしい言葉遣いになっているか（「沖止め」「干し上がり」等の現地語）

**緊急アラートの文言:**
- [ ] 「⚠️ 急変」「天気が崩れます」など緊急性が伝わる表現か
- [ ] 変化量（例: 「スコアが80→45に低下」）が含まれるか
- [ ] 「すぐに昆布を取り込んでください」等の行動指示があるか

### B. チャットボット返答のLINE転送適性

- [ ] 「今日の○○部落の予報を教えて」への返答がコピペしてLINEに送れる形式か
- [ ] 返答文が箇条書きの場合、「・」や改行がLINEで崩れないか
- [ ] 漢字・ひらがな・カタカナのバランスが読みやすいか（難読漢字を避けているか）
- [ ] 部落名の誤字（例「鬼脇」「鴛泊」「仙法志」）がなく正確か

### C. 干場カード情報のLINE共有適性

- [ ] 干場名・スコア・判定が1スクリーンショットに収まるか
- [ ] 数字に単位が付いているか（「風速 3.2m/s」「湿度 78%」「気温 12.5°C」）
- [ ] 「H_1631_1434」のような内部コードが漁師さん向けに部落名で補完されているか

### D. 文章品質の全般チェック

- [ ] 主語・述語が明確か（「干せます」「干せません」を明言しているか）
- [ ] 否定表現が婉曲すぎないか（「干しにくい可能性があります」→「干せません」の方が明快）
- [ ] 敬語レベルが統一されているか（丁寧体「です・ます」調か）
- [ ] 「予測」「予報」「見込み」などの言葉が正しく使い分けられているか
- [ ] 誤字・脱字・変換ミスがないか（特に地名：利尻・沓形・鴛泊・仙法志など）

### E. LINEテキスト機能の便利さ（UX評価）

- [ ] 「今日の全体予報をテキストでコピー」ボタン等の**クイック共有機能**があるか
- [ ] なければ：テキストコピー機能の追加を 🟡中 の改善として提案する
- [ ] LINEのShare APIやURLスキームで直接LINEに送信できる機能があるか
- [ ] なければ：追加を 🟢低 の改善として提案する
