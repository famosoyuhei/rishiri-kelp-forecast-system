## チェックリスト

### A. 乾燥判定の絶対条件（3条件）

- [ ] **降水量 = 0mm** が絶対条件として実装されているか（`precipitation == 0`、`< 1` ではない）  
  ※ CLAUDE.mdのK8項目: 「0.5mm雨でも昆布干し不可」のため `< 1` は不可
- [ ] **最低湿度 ≤ 94%** が正しく参照されているか（最低値であり平均値ではない）
- [ ] **平均風速 ≥ 2.0 m/s** が正しく実装されているか
- [ ] 3条件すべてを満たさない場合、スコアが「干し不適（0〜20点相当）」になるか

### B. 地形補正の実装（start.py L847-L863付近）

**風速補正（加算型）:**
- [ ] `is_forest == True` のとき `wind_speed -= 2.5` が実装されているか
- [ ] `is_coastal == True` のとき `wind_speed += 1.0` が実装されているか
- [ ] 補正後のwind_speedが負の値にならないようガードされているか（`max(0, wind_speed)` 等）

**湿度補正（固定値）:**
- [ ] `is_forest == True` のとき `humidity += 10.0` が実装されているか
- [ ] `is_coastal == True` のとき `humidity += 5.0` が実装されているか
- [ ] `elevation > 10` のとき `humidity -= elevation / 100 * 1.0` が実装されているか
- [ ] 補正後のhumidityが0〜100の範囲にクリッピングされているか

**気温補正:**
- [ ] 気温の独自補正が**削除されている**か（Open-Meteoが0.7°C/100mで自動補正済み）

### C. スコア計算ロジック（calculate_enhanced_drying_score）

- [ ] スコアの出力範囲が0〜100点に正規化されているか
- [ ] 日射量（`avg_solar_radiation`）がスコアに**影響を与えているか**  
  ※ CLAUDE.mdのK1: 日射≥400W/m²で+15、<50W/m²で-15の実装状況を確認
- [ ] 曇天時（日射<50W/m²）の solar_score が 20→0 に修正されているか（K4）
- [ ] スコアが「◎ 85点以上」「○ 70〜84点」「△ 50〜69点」「× 49点以下」などに分類されているか

### D. 特別地点の予報データ

- [ ] `A_1783_1383`（沓形アメダス）の予報が取得できるか
- [ ] `A_2417_1867`（本泊アメダス）の予報が取得できるか
- [ ] `R_1800_2392`（利尻山頂 1,721m）の予報が取得できるか
- [ ] 山頂地点で風速・湿度補正が地上付近と異なる値になっているか（標高効果）

### E. Open-Meteo APIパラメータの整合性

- [ ] `elevation` パラメータが全API呼び出しに追加されているか
- [ ] `dewpoint_2m` が取得されているか（霧リスク判定の前提データ）
- [ ] `shortwave_radiation` が取得されているか（`direct_radiation` の上位互換）
- [ ] JSTタイムゾーン（UTC+9）でデータが正しく解釈されているか（UTC誤差チェック）

### F. 未実装機能の確認（ロードマップ項目）

以下は現時点で未実装とされているが、**誤実装がないかを確認**してください：
- [ ] 霧リスクスコア（`(temp - dewpoint) < 2°C`）→ 未実装のまま放置されているか（誤った実装がないか）
- [ ] フロード数診断 → 未実装のまま放置されているか
- [ ] CAPE指数 → 未実装のまま放置されているか
