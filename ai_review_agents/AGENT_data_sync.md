## チェックリスト

### A. 4ファイルの現在の整合性確認

- [ ] `hoshiba_spots.csv` の地点数と `all_spots_array.js` の地点数が一致するか
- [ ] `hoshiba_spots_named.kml` の地点数がCSVと一致するか
- [ ] `hoshiba_records.csv` に特別地点（A_/R_）の記録が含まれていないか（干場のみ）
- [ ] 3特別地点（A_1783_1383、A_2417_1867、R_1800_2392）がCSV・KML・JSに存在するか

### B. 干場追加（`POST /add`）の同期

- [ ] 地図クリック → 座標取得 → `H_XXXX_XXXX` 命名が正しく実行されるか
- [ ] 追加後に4ファイル全てに新干場が追加されるか（同期の完全性）
- [ ] KMLファイルのXML構造が破壊されないか（malformed XMLにならないか）
- [ ] JSファイルが正しいJavaScript配列構文で書き出されるか（JSON構文エラーがないか）
- [ ] 追加後に `GET /api/spots` を呼び出すと新干場が含まれるか

### C. 干場削除（`POST /delete`）の5条件制限

以下の5条件が**すべてコードで検証されているか**確認してください（start.py:985-996）：
- [ ] ① 乾燥記録（hoshiba_records.csv）が存在しない
- [ ] ② お気に入り登録されていない
- [ ] ③ 通知設定で使用されていない
- [ ] ④ 同時編集ロック（5分間）がかかっていない
- [ ] ⑤ 機械学習の訓練データとして使用されていない

**特別地点の保護:**
- [ ] `A_` または `R_` で始まる地点は削除リクエストを拒否するか
- [ ] 拒否時のエラーメッセージが「システム保護地点は削除できません」等と明示されているか

### D. 乾燥記録（`POST /record` / `GET /record/<name>/<date>`）

- [ ] `H_` で始まる干場のみ記録が追加できるか（特別地点は拒否されるか）
- [ ] 記録追加後に `hoshiba_records.csv` に追記されるか
- [ ] 同一干場・同一日付の記録を更新した場合、重複行にならないか（上書きされるか）
- [ ] `GET /record/<name>/<date>` が存在しない記録に404を返すか

### E. オフライン時の仮保存と自動同期

- [ ] オフライン時に記録操作が「仮保存」としてlocalStorageに保存されるか
- [ ] オンライン復帰後に仮保存データが自動でサーバーに送信されるか
- [ ] 自動同期に失敗した場合に再試行ロジックがあるか
- [ ] 自動同期後に仮保存データが正しく削除されるか（二重送信防止）

### F. ファイルバックアップ

- [ ] `hoshiba_records.csv` のバックアップが毎日実行される設定があるか
- [ ] `hoshiba_spots.csv` の変更時バックアップが実装されているか
- [ ] Renderデプロイ時にデータファイルが上書きされないか（永続ストレージの確認）
