## チェックリスト

### A. レスポンス速度（性能）

- [ ] `GET /api/forecast`（334地点×7日間）が **10秒以内** に完了するか
- [ ] `GET /api/spots` が **2秒以内** に完了するか
- [ ] 初回ページロードが **5秒以内** に完了するか（Renderのコールドスタート考慮）
- [ ] Open-Meteo APIへのリクエストに **タイムアウト設定（例: 15秒）** があるか
- [ ] 334地点の予報を並列リクエストで取得しているか（逐次処理では遅すぎる）
- [ ] 予報データのキャッシュ（例: 1時間キャッシュ）が実装されているか

### B. 入力バリデーション

**`POST /add`（干場追加）:**
- [ ] 緯度が 45.0〜45.5 の範囲外を拒否するか（利尻島の範囲外座標防止）
- [ ] 経度が 141.0〜141.5 の範囲外を拒否するか
- [ ] リクエストボディが空の場合に400を返すか

**`POST /delete`:**
- [ ] `spot_name` に `../` や `%2F` 等のパストラバーサル文字が含まれていないか検証しているか
- [ ] `spot_name` が `H_XXXX_XXXX` 形式に一致しない場合に400を返すか

**`POST /record`:**
- [ ] `date` フィールドが有効な日付形式（YYYY-MM-DD）か検証しているか
- [ ] `score` フィールドが0〜100の数値範囲か検証しているか
- [ ] XSS（クロスサイトスクリプティング）対策：テキストフィールドにHTMLタグを入力しても無害化されるか

### C. CORS設定

- [ ] CORSの許可オリジンが `*`（全許可）になっていないか
- [ ] 許可オリジンが `https://rishiri-kelp-forecast-system.onrender.com` に限定されているか
- [ ] 開発時のローカルオリジン（`http://localhost:XXXX`）が本番設定から除外されているか

### D. 情報漏洩・シークレット管理

- [ ] APIキー・秘密鍵がコード（`start.py`）にハードコードされていないか
- [ ] `.env` ファイルが `.gitignore` に含まれているか
- [ ] `hoshiba_records.csv` や `hoshiba_spots.csv` が直接URLアクセスで取得できないか
- [ ] Renderのログに個人情報（メールアドレス等）が出力されていないか
- [ ] GitHub公開リポジトリに秘密情報がコミットされていないか（git log確認）

### E. Renderデプロイの安定性

- [ ] Renderのスリープ対策として `/health` エンドポイントへの定期pingが設定されているか（または有料プラン利用）
- [ ] `gunicorn` のワーカー数が適切か（`Procfile` 確認）
- [ ] メモリ使用量が512MB以下か（Render無料プランの制限）
- [ ] matplotlibの削除によるメモリ削減が確認されているか（旧依存関係が残っていないか）

### F. フロントエンドセキュリティ

- [ ] `innerHTML` に外部データを直接代入しているコードがないか（XSS原因）
- [ ] `eval()` の使用がないか
- [ ] localStorageに保存されているデータに個人情報・機密情報が含まれていないか
- [ ] 通知の免責事項同意がbypassされないか（localStorageを手動削除しても再表示されるか）
