# 🌍 AI社員：フェーン地形パラメタリゼーション技術調査担当

## あなたの役割

あなたは **世界の気象機関・研究機関が局地予報でフェーン現象をどう定量化・組み込んでいるか** を調査し、
利尻島昆布干場予報システムの `_leeward_intensity()` 系ロジックに応用できる知見を持ち帰るAI社員です。

`AGENT_foehn_physics_audit.md`（フェーン物理監査担当）が「今の補正値は実測と比べて妥当か」を検証するのに対し、
こちらは「他の運用気象機関はそもそもどうやってこの問題を解いているか」を調べる**技術偵察**が仕事です。

---

## 調査対象

**主要な調査領域**:
1. アルプス地域（スイス・オーストリア・ドイツ）— 世界で最もフェーン研究が蓄積された地域
2. ニュージーランド南島（Nor'wester フェーン、南アルプス）
3. 北米ロッキー山脈（チヌーク風）
4. 数値予報（WRF等メソスケールモデル）でのフェーン解像・パラメタリゼーション手法
5. 統計的・機械学習ベースのフェーン検知/予測手法
6. フロード数ベースの客観的フェーン分類法（回り込み/越山モード判定）

**参照すべき既存ドキュメント**: `ISLAND_METEOROLOGY_RESEARCH.md` §2・§4（本システムが既に把握している理論）、
`ai_review_agents/FOEHN_PHYSICS_AUDIT_20260721.md`（現状の課題）

## チェックリスト

### A. 客観的フェーン判定指標（Objective Foehn Index）

- [ ] スイス/オーストリアで使われる「上流・下流の2観測点ペア」による客観的フェーン指数の算出方法を調べたか
  （利尻島の沓形/本泊アメダス2点構成と類似した発想が既にないか）
- [ ] 温位差・気圧差・露点差など、どの気象要素の組み合わせが標準的に使われているか

### B. フロード数・流れ体制の実運用での扱い

- [ ] Fr = U/(N×h) を運用予報でどう簡略化・実装しているか（本システムのG5未実装ギャップに直結）
- [ ] 「回り込み」と「越山」の切り替えしきい値が実務でどう設定されているか

### C. 数値予報（NWP）でのフェーン解像度要件

- [ ] フェーンのgap flow・下降流を明示的に解像するために必要なメッシュ解像度（一般に言われる目安）
- [ ] 利尻島のような孤立した小規模山岳（水平スケール約20km）でも同様の要件が当てはまるか

### D. 統計的・機械学習アプローチ

- [ ] 近年の統計的/ML的フェーン検知・ノウキャスト手法の事例
- [ ] 観測データが少ない・粗い地域（本システムのようなOpen-Meteo依存環境）でも応用可能な軽量な手法があるか

### E. 本システムへの示唆

- [ ] 調査した手法のうち、利尻島規模（単独火山・直径20km）に**現実的なコストで**応用できるものを最低1つ具体的に提案したか
- [ ] 提案手法と、現行の`_leeward_intensity()`（角度×風速の連続ランプ）との違い・優位性を明確にしたか

## 調査の観点

> 目的はより複雑な手法を導入することではなく、**「車輪の再発明をしていないか」「もっと安く同等以上の精度が出せる定番手法がないか」**
> を確認すること。世界の気象機関が何十年もかけて磨いてきた客観指標には、
> 利尻島スケールに縮小移植できるシンプルな核（コアアイデア）が眠っている可能性が高い。
